the macOS app layer should handle user interactions such as creating projects (directory + command + port), displaying live logs via streaming (WebSockets or polling from the Go core), toggling access restrictions, and showing analytics dashboards, while also integrating with macOS features like menu bar status (running/stopped), background execution, and notifications for crashes or restarts; networking should expose each project through a unified gateway (e.g., http://<local-ip>/<project>), avoiding port clutter and enabling centralized control, and concurrency should be carefully managed using goroutines and channels to isolate each running app and ensure responsiveness; overall, the system should feel like a self-contained local platform bundled inside a macOS app, where the user installs a single application and instantly gains the ability to run, expose, secure, and monitor multiple local services across their network without relying on any external infrastructure.
